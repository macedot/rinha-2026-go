# Agent Instructions

## Architecture Reality Check

- **CGo is removed:** Ignore the `README.md` claims about C/AVX2 and the `bridge` submodule. The project has been ported to pure Go (`CGO_ENABLED=0` in `Dockerfile`). The IVF vector search engine is in `internal/ivfsearch/ivfsearch.go` with AVX2 assembly in `internal/ivfsearch/ivfsearch_amd64.s`.
- **No fasthttp:** The server uses a raw `epoll` event loop via `golang.org/x/sys/unix` — not `fasthttp` or `net/http`. Connections are handled synchronously in a single thread.
- **Zero-alloc JSON:** The `/fraud-score` endpoint uses a custom, zero-allocation sequential parser in `internal/vectorizer/vectorizer.go` instead of `encoding/json`. It assumes a fixed key order in the payload. Maintain this pattern for the hot path.
- **UDS FD Passing:** The API does not expose standard TCP ports. It receives connections via UNIX Domain Sockets (UDS) using `SCM_RIGHTS` file descriptor passing from a custom load balancer (`SoNoForevis`). See the epoll loop in `cmd/server/main.go`.
- **CPU Pinning:** The server pins itself to a single CPU core based on a hash of the UDS path. Combined with `GOMAXPROCS=1` and `GOGC=off` in `docker-compose.yml`.

## Developer Workflow

- **Entrypoints:**
  - `cmd/server/main.go`: API server (raw epoll loop).
  - `cmd/build_index/main.go`: Generates the IVF index from the reference dataset.
  - `cmd/bench/main.go`: Search latency benchmark.
- **Local Setup:** Running `cmd/server` or `cmd/bench` locally without Docker requires the index to be present. Build it first:
  ```bash
  go run ./cmd/build_index resources/references.json.gz resources/index.bin
  ```
- **Assembly Generation:** AVX2 assembly is generated by `avo` (removed from module after generation). To regenerate, you'd need to temporarily add `github.com/mmcloughlin/avo` and run the generator.
